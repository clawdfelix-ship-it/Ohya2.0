# 香港獨立站電商後台（Full Ops Backoffice）設計規格

## 背景與目標

本專案已具備前台購物流程（EJS）及一組以 session + `is_admin` 控制的管理端 API（`/api/admin/*`），同時已存在 `/admin/*` 後台頁面骨架（EJS + 少量前端 JS fetch JSON）。現階段後台仍偏向「JSON viewer」，未足以支援日常營運。

本規格目標是把後台擴展成「可落地營運」的香港獨立站電商管理系統，覆蓋六大核心維度：
- 交易閉環
- 日常營運
- 會員復購
- 數據驅動
- 合規安全
- 本地場景

同時要求：保持現有架構（Node + Express + EJS + session），先以可落地 MVP 跑通交易閉環，再逐步補齊規模化營運能力。

## 範圍（Scope）

### In Scope

- 建立可營運級後台資訊架構（IA）：商品/庫存、訂單/履約、會員、營銷、內容/SEO、報表、合規安全、系統設定。
- 補齊「狀態機 + 必備欄位 + 操作日誌 + 審批」四大後台底座能力。
- 香港本地化：物流（順豐/香港郵政/嘉里、自提/智能櫃/到店自取）、地址校驗、WhatsApp 通知、FPS/PayMe 支付場景、節假日提示。
- 以現有 `/api/admin/*` 為基礎統一管理端 API，逐步消除重複路由與行為不一致。

### Out of Scope（第一版不做）

- 全新 SPA 後台（React/Vue）與複雜前端構建流程。
- 企業級 IAM（完整 RBAC 權限表、SSO、IP allowlist、審計報表可視化）。
- 深度 BI（自助建模、數據倉庫、即席查詢引擎）。

## 角色與權限（最小可用版本）

### 角色

- 超級管理員：全權（含系統設定、權限、整合）
- 營運：商品/分類/活動/內容、訂單處理（不含退款打款）
- 倉務：揀貨/出貨/庫存、退貨入庫驗貨
- 客服：工單/售後、訂單備註、訊息回覆（不能改價/退款）
- 財務：對賬/退款審批/報表導出、會計整合

### 權限粒度（後續可擴展）

最小可用採用「角色固定權限」，功能點粒度到：
- 讀取（view）
- 編輯（edit）
- 批核（approve）
- 導出（export）

## 核心狀態機（必須統一字典）

### 訂單狀態（Order Status）

- `pending_payment`（待付款）
- `paid`（已付款/待處理）
- `processing`（揀貨中/處理中）
- `shipped`（已出貨）
- `completed`（已完成）
- `cancelled`（已取消）
- `after_sales`（售後中）

### 支付狀態（Payment Status）

- `unpaid`、`authorizing`（如信用卡）、`paid`、`failed`、`refunding`、`refunded`、`partial_refunded`

### 履約狀態（Fulfillment Status）

- `unfulfilled`、`partially_fulfilled`、`fulfilled`、`returned`

### 退款/退貨狀態（After-sales）

- 退款（Refund）：`requested` → `approved` → `processing` → `refunded` / `rejected`
- 退貨（Return）：`requested` → `approved` → `in_transit` → `received` → `inspected` → `refunded` / `replaced` / `rejected`

## 交易閉環（端到端流程）

### 下單 → 付款 → 鎖庫 → 出貨 → 簽收/完成

- 下單：
  - 建立訂單、訂單項、收貨地址快照、價格拆分（商品/運費/折扣/手續費）
  - 若用到優惠：寫入「優惠使用記錄」
- 付款：
  - 支付成功 webhook 更新 `payment_status=paid`，同時觸發鎖庫
  - FPS/PayMe 場景支援「上傳付款證明 + 人工核實」，核實後轉 `paid`
- 鎖庫：
  - 付款成功即鎖庫（從可售庫存扣到已鎖庫存），取消/退款要自動釋放
- 出貨：
  - 倉務揀貨→包裝→打印面單→出貨
  - 生成物流單號 + 承運商 + 追蹤狀態
- 完成：
  - 物流簽收或超時自動完成（可配置）
  - 完成後觸發：發票/憑證歸檔、評價邀請、積分入賬（如有）

### 售後 → 退款/退換貨 → 入庫驗貨 → 財務對賬

- 客服建立/受理工單，綁定訂單與會員
- 退款需走審批（財務/超級管理員），支援全額/部分退款
- 退貨入庫需走驗貨流程（倉務），可記錄相片與檢驗結果
- 原路退款優先；FPS/PayMe 手動退款需記錄退款憑證與操作人

## 功能模塊（後台 IA 與必備功能）

### 1. 商品與庫存

- 商品：新增/編輯/上下架/定時上下架/回收站/審批
- SKU：規格組合、條碼/貨號、成本價/售價、每 SKU 庫存
- 媒體：主圖/詳情圖/短片，資源庫統一管理
- 定價：原價/促銷價/會員價/限時價、會員等級價（第二階段可做階梯價/批發價）
- 庫存：
  - 多倉庫
  - 入庫/出庫/調撥/盤點
  - 庫存流水（每次變動可追溯到訂單/採購/盤點）
  - 低庫存預警
- 批次/效期（可選）：批次號、到期日、到期預警
- SEO：商品 meta、slug、OpenGraph
- 評價：審核/回覆/置頂/屏蔽

### 2. 訂單與履約

- 訂單列表：搜尋/篩選（狀態、日期、支付方式、物流商、地區）、批量導出
- 訂單操作：改備註、改地址（未出貨）、改價（需審批）、取消（需規則）
- 詳情頁：訂單快照（商品/地址/金額拆分/優惠/物流/發票/操作日誌）
- 出貨：揀貨單、部分出貨、批量出貨、物流狀態同步
- 異常：支付失敗、庫存不足、可疑訂單提示、COD 訂單專屬流程（如啟用）
- 香港配送：
  - 承運商：順豐香港、香港郵政、嘉里（可擴展）
  - 配送方式：上門、智能櫃、自提點、到店自取
  - 運費模板：港島/九龍/新界/離島＋偏遠附加費

### 3. 會員與客戶營運

- 登入：電郵/手機、社交登入（可選）、WhatsApp 綁定狀態
- 會員檔案：地址簿、訂單/消費/退款記錄、標籤、同意/退訂記錄
- 會員等級：升級條件、權益（折扣/免運/生日禮/優先出貨）
- 積分（可選）：入賬/扣回、有效期、用途（抵扣/換券/換禮）
- 分群：新客/回頭客/高消費/沉睡/棄單，支援保存分群條件

### 4. 營銷與推廣

- 優惠券/優惠碼：門檻、有效期、疊加規則、核銷記錄
- 活動：限時特價、滿額贈、套裝（先做最小可用）
- 棄單召回：識別、WhatsApp/電郵觸達、頻率控制、效果統計
- 像素/追蹤：GA4、Meta Pixel、TikTok Pixel、Merchant Center（事件口徑一致）
- 分銷聯盟（第二階段）：推廣員、傭金、結算、退貨扣回

### 5. 財務與對賬

- 交易流水：按支付渠道、幣種、日結/月結
- 對賬：支付流水 vs 訂單差異
- 退款：原路退款、手動退款憑證、審批與日誌
- 憑證/發票：電子發票（可選）、憑證歸檔與 7 年留存策略
- 會計整合（第二階段）：Xero/MYOB 匯出或 API 推送

### 6. 報表與數據

- 總覽儀表板：營業額、訂單量、客單價、轉化率（如接 GA4）、復購率、退款率
- 商品：熱銷/滯銷、庫存周轉、低庫存
- 會員：新客、回頭客、沉睡客
- 營銷：券核銷率、棄單召回成功率
- 導出：CSV（欄位口徑固定、含時區 HK）

### 7. 合規與安全

- PDPO：資料收集聲明、用戶資料匯出/刪除申請處理、Cookie 同意記錄
- 垃圾訊息：營銷退訂、同意記錄留存、發送頻率控制
- 後台安全：2FA（管理端第二階段）、密碼強度、登入/操作日誌
- 風控：可疑訂單提示、黑名單、異常登入提醒（最小可用）

## 管理端 API 統一原則（與現有專案對齊）

- 後台頁面（EJS）只做渲染與最少 JS，資料與權限驗證集中在 `/api/admin/*`。
- 同一路徑只允許一份實作，避免重複 route 造成行為不一致。
- API 回傳格式統一：`{ ok, data, error, meta }`（逐步改造，不要求一次過重寫全部）

## 與現況對照（已具備 vs 缺口）

### 已具備

- `/admin/*` 後台頁面骨架（Dashboard/Orders/Products/Categories/Users）與 login/setup。
- `/api/admin/orders`（list/detail/status）與部分 dashboard/products/users 管理端 endpoint。

### 主要缺口（阻礙落地營運）

- 後台 UI 未有「可操作」表格/表單（目前以 JSON `<pre>` 呈現）。
- `/api/admin/users`、products 相關 endpoint 存在多份重複定義，路由載入順序會令較完整版本無法生效。
- 缺少一致的狀態字典、操作日誌、退款/售後/對賬等閉環模塊。

## 分階段交付（落地優先級）

### MVP（開店即用：跑通交易閉環）

- 訂單：列表/詳情/更新狀態、出貨、物流單號、通知
- 商品：列表/新增/編輯/上下架、基本庫存、低庫存預警
- 分類：基本增刪改
- 會員：列表/詳情、凍結/黑名單（最小）
- 支付：支付狀態同步與手動核實（FPS/PayMe）
- 基礎報表：日成交、訂單量、低庫存
- 操作日誌：訂單/商品關鍵操作可追溯

### 第二階段（90 日內：提升效率與復購）

- 售後工單＋退款審批＋退貨入庫驗貨
- 優惠券/活動（最常用）、棄單召回（WhatsApp 優先）
- 會員等級＋生日禮/回購券
- 對賬報表＋憑證歸檔（7 年留存）
- 香港地址校驗＋分區運費模板＋自提/到店自取流程完善

### 第三階段（規模化：供應鏈、分銷、深度分析）

- 採購/供應商/多倉調撥/盤點
- 分銷聯盟、傭金結算
- 深度 BI、角色權限表（細粒度 RBAC）、2FA 全面化

## 驗收標準（Acceptance Criteria）

- 管理員可用後台完成：上架商品 → 產生訂單 → 確認付款 → 自動鎖庫 → 出貨 → 查物流 → 完成訂單。
- 後台可用 UI 完成最小操作：新增/編輯商品、調整庫存、更新訂單狀態、出貨錄入單號、導出訂單。
- 所有關鍵操作均有操作日誌；退款/改價等高風險操作有審批或權限限制。
- 香港本地化流程可用：至少支援一個本地物流方式與 WhatsApp 通知（或預留整合位）。
